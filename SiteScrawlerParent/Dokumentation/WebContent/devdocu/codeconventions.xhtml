# Code Conventions ## Kommentare Generell gilt der Grundsatz, dass der Code so kommentiert werden muss, dass er ohne andersweitige Erklärung verständlich sein muss. Allerdings müssen Verhaltensweisen die durch die Code bereits klar erkennbar sind nicht weiter kommentiert werden. Es gelten folgende Regeln: * Öffentliche Methoden und Attribute müssen ausnahmslos kommentiert werden. * Code muss inline kommentiert werden wenn die Intention des Codes nicht klar erkennbar ist * Kommentare müssen wärend des Programmierens laufend erstellt werden, nicht erst nach dem der Code fertig gestellt ist. * Der Code muss vollständig kommentiert sein, bevor er in den Main Branch aufgenommen wird. Falls Kommentare fehlen wird der Pull-Request abgelehnt ## Richtlinien für Clean Code Der Code sollte klar strukturiert sein, nach Funktion in Methoden und Klassen aufgeteilt werden und Methoden sowie Attributnamen sollten aussagekrätig benannt werden. * Der Code sollte nach Funktionen aufgeteilt werden, es gilt: Jede Funktion die als eigene Methode ausgelagert werden kann sollte auch ausgelagert werden. Jede Klasse sollte nur eine Aufgabe übernehmen.
